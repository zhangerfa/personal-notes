# 二发的个人笔记 (Erfa's Personal Notes)

这是一个基于 AI 驱动的个人知识管理仓库，由 **Agent** 辅助记录与维护。本仓库旨在沉淀技术洞察、哲学思考以及日常生活中的灵光一现。

## 📁 目录结构

仓库采用结构化的分类管理方式，并通过 `INDEX.md` 维护全局索引：

- **[技术与工程](./技术与工程/)**：记录关于 AI、工程架构、Skill 开发等技术领域的深度思考与实践。
- **[哲学与思考](./哲学与思考/)**：沉淀关于人生、社会、自我认知的哲学探索。
- **[个人随笔](./个人随笔/)**：记录日常生活中的感悟与随笔。
- **[金融与投资](./金融与投资/)**：分析市场逻辑、投资框架与宏观经济。
- **[INDEX.md](./INDEX.md)**：全局索引文件，记录所有笔记的路径及其核心条目摘要。
- **[skill_docs/](./skill_docs/)**：存放 AI 助手工作流相关的官方说明文档。

## 🤖 工作流与 Skill 机制

本仓库的核心管理能力由 `personal-notes` 技能提供。

### 触发机制
在与 Agent 对话时，通过以下方式即可触发笔记记录：
- 明确使用触发词：“记下来”、“补充笔记”、“加到笔记”。
- 以“**笔记：**”开头发送内容。
- 发送具有记录价值的个人观点或深度思考。

### GitHub PR 审查工作流（核心原则）
为了保证笔记内容的准确性与可控性，同时提升效率，Agent 遵循以下流程：
1. **自动分支提交**：当笔记发生新增或修改时，Agent 会自动创建一个以 `note-update-` 为前缀的新分支并完成本地 Commit。
2. **推送并创建 PR**：Agent 将新分支推送到 GitHub 远端（`github.com/zhangerfa/personal-notes.git`），并向用户提供 **GitHub Pull Request (PR) 创建链接**。
3. **用户 Review**：由用户在 GitHub 网页端进行代码审查和合并，合并后即完成笔记的正式归档。

## 🌍 环境与部署

本仓库及其配套的 Agent 助手支持跨环境部署，能够自动适应不同的运行上下文：

- **环境感知的路径解析**：
  Agent 会根据当前运行上下文自动推断笔记的物理存储路径：
  1. **云端沙盒 (Workspace)**：若检测到处于 Aime 云端沙盒环境（如 `WORKSPACE_ID` 存在），默认路径为 `./memory/notes`。
  2. **本地物理机**：若在用户个人电脑运行，默认路径回退至家目录下的 `~/memory/notes`。
