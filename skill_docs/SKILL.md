# Personal Notes Skill

## 功能

记录用户的想法、观点、哲学思考到 `memory/notes` 文件夹下。

## 触发条件

当用户发送以下内容时使用此 skill：
- 明确说"补充笔记"、"记下来"、"加到笔记"
- 消息以"笔记："开头
- 发送一段个人观点、想法、哲学思考，并期望被整理记录

## 全局索引与双段式加载

- 在 `memory/notes/` 根目录下必须维护一个全局索引文件 `INDEX.md`。
- **`INDEX.md` 的结构**：按文件夹和文件层级，记录所有笔记文件的路径，以及该文件包含的核心条目清单（目录/摘要词）。
- **双段式加载与查重逻辑**：
  1. **第一阶段（轻量检索）**：接收新笔记后，Agent **仅读取** `INDEX.md`，判断新内容可能所属的分类或文件路径。
  2. **第二阶段（深度比对）**：锁定潜在目标文件后，定向读取该具体文件的完整内容，进行深度语义比对（判断是追加还是合并修改）。

## 索引缺失的处理逻辑

- 如果发现 `INDEX.md` 不存在或已失效，必须先触发一次全局扫描（遍历 `memory/notes/` 下的所有 `.md` 文件），重新生成 `INDEX.md`，然后再处理用户的新笔记。

## 核心规则

### 1. 写入与反馈（强制确认流程）

每次更新笔记，必须严格遵循以下“展示-确认-写入”的三步法流程，**此流程为不可逾越的硬性规定**：

1.  **展示 Diff（强制拦截）**：在分析完合并逻辑并生成 Git Diff 后，Agent **必须立即中断任务执行**，将完整的 Diff 内容清晰地打印给用户。
2.  **阻塞等待**：**严禁在展示 Diff 的同时调用任何写入文件的工具（如 `write_file` 或 `search_replace_file`）**。Agent 必须在此阶段停止，明确询问用户是否确认执行修改。
3.  **授权写入**：只有当用户明确回复“同意”、“确认”、“可以”、“OK”等许可指令后，才能执行真正的本地文件写入操作（包括笔记文件和 `INDEX.md` 索引文件）。

**特别说明**：完成笔记文件的写入后，**必须同步更新 `INDEX.md`** 中的相关目录或摘要，确保索引与实际内容一致。

### 2. 禁止 Push
本技能的所有操作仅限于本地文件的创建与修改，**绝对禁止调用 `git push` 命令**。

### 3. 无关键词
整理后的笔记不添加"关键词"标签。

### 4. 同步机制
更改完不要立即同步 git，会有定时任务每天十点同步笔记到 git 仓库中。

## 笔记文件

- 默认目录：`memory/notes`
- 默认文件：`memory/notes/personal-notes.md`（或根据分类自动选择/创建文件）
- **自动处理**：如果目录或文件不存在，应自动创建。新文件初始内容为 `# Personal Notes`。

## 格式规范

每条笔记作为独立的二级标题（`## 标题`），标题下注明日期（`> YYYY-MM-DD`），内容适当整理排版，保持简洁清晰格式。

## 去重规则

- **语义合并**：写入前扫描已有笔记，如果存在语义高度相似的条目，不新建条目，而是将新内容合并到已有条目下，允许删改已有笔记。
- **展示标注**：在展示 diff 时明确标注 "检测到与已有笔记「XXX」相似，已合并"。

## 分类与检索

- **主题分组**：当笔记条目超过 20 条时，自动按主题分组，使用文件夹分类，并在对应文件内使用一级标题（#）作为分类，二级标题（##）作为具体笔记。分类示例：`# 技术思考`、`# 人生哲学`、`# 工作反思`。
- **目录索引 (TOC)**：在每个文件顶部维护一个目录索引，格式为 `- [标题](#锚点)`，方便快速跳转。
- **分类逻辑**：由 skill 根据内容语义自动判断分类，用户也可以指定。

## 文件分片

- **上限限制**：单文件上限 500 条笔记或 50KB（以先到者为准）。
- **自动分片**：达到上限后自动按语义拆分为多个文件或文件夹。
