# Personal Notes Skill

## 定位

本 skill 专门处理**碎片化的个人想法**——生活感悟、价值观、随机观察，而不是系统性技术笔记。
核心任务是：**接收零散想法 → 找到归宿 → 与已有内容融合提炼 → 呈现给用户确认 → 提交**。

## 工作流程

### Step 1：定位归宿（轻量检索）

读取仓库根目录的 `INDEX.md`，判断新想法属于哪个分类和文件。  
若 `INDEX.md` 不存在或明显过时，先扫描所有 `.md` 文件重建索引。

### Step 2：读取目标文件，寻找融合点

读取目标文件全部内容，按以下三档判断处理方式：

| 情况 | 处理方式 |
|------|---------|
| 新想法与某条目**主题相同** | **合并**：将两者整合为更丰富、更统一的表述 |
| 新想法与某条目**相关但侧重不同** | **相邻插入**：在相关条目附近新建，保持语义连贯 |
| 新想法与现有内容**无关联** | **新建条目**：选择语义最近的位置插入 |

**改写约束**：整合时保留原有核心观点，不得删除原有实质内容；只在表达更清晰或与新内容自然融合时才改写标题和措辞。

### Step 3：呈现变更，等待用户确认

整合完成后，**不要直接提交**，先向用户展示变更内容：

```
📝 准备写入「<文件名>」

【变更前】
<原条目内容，若为新建则写"（新建条目）">

【变更后】
<整合后的新内容>

确认提交？
```

只展示受影响的条目片段，不打印完整文件或 git diff。

### Step 4：用户确认后，提交并推送

```bash
git checkout -b note-update-$(date +%Y%m%d%H%M%S)
git add .
git commit -m "feat: add/update note"
git push codebase <branch>
git push github <branch>
```

推送完成后反馈 MR/PR 链接，同步更新 `INDEX.md`。

## 笔记仓库路径

- **本地物理机**：`~/erfa/personal-notes`
- **云端沙盒**（检测到 `WORKSPACE_ID`）：`./memory/notes`

执行前通过 `git remote -v` 确认远端 `codebase`（主）和 `github`（备）均已配置。

## 格式规范

```markdown
## <核心观点作为标题>

> YYYY-MM-DD

<内容，口语化、简洁，读起来像在对自己说话>
```

**标题规范**：
- 用核心观点做标题，不用引用来源（不写"荣格说"、"某研究表明"）
- 过于简短、无法独立成条的想法，优先融入已有条目，不单独立标题

## 分类原则

仓库结构（参考 `INDEX.md`）：
- `哲学与思考/` — 人生观、价值判断、哲学感悟
- `个人随笔/` — 生活经验、自我认知、关系与社交
- `技术与工程/` — 技术方法论、工程思考（非系统知识，是想法层面）
- `金融与投资/` — 财务认知、投资逻辑

内容超过 20 条时自动按主题在文件内分组；单文件超过 50KB 时拆分。
