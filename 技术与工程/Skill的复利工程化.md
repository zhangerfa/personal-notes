# Skill 的复利：把隐性经验外化成可迁移的工程载体

> 2026-03-27

- Skill 不是长 Prompt 的包装，而是将隐性工作经验工程化成**可复用、可迁移、可复利**的载体，解决“Agent 从能用到好用”的最后一公里。
- **渐进式披露**是关键：按需加载（元数据 → 主体 → 引用/脚本），避免“越全面越污染上下文、越容易过度分析”。
- Skill 的价值更像**单元测试**：第一次构建更贵，但从第二次开始回本，并能跨人/跨团队复用。
- “**用最贵模型设计 Skill，用最便宜模型执行 Skill**”是一种工程蒸馏：强模型沉淀排查路径/异常分支/边界，轻量模型负责日常执行。
- 好 Skill 的重点是**定义边界与关键岔路口的人工拍板**，而不是把每一步写死成脚本。
- MCP 是工具箱，Skill 是菜谱：工具越多越需要方法约束。
